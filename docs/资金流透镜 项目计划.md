# 资金流透镜 项目计划

## 项目定位

`资金流透镜` 是一个个人长期可用的 A 股资金流统计网页版。

核心目标不是做行情终端，而是围绕“股票 / 板块 + 时间区间”稳定完成资金流数据拉取、缓存、统计、可视化和导出。

- 仓库名：`stock-flow-lens`
- 产品名：`资金流透镜`
- 使用场景：本地部署，个人使用
- 第一阶段目标：完成可用的数据查询、统计、缓存和展示闭环
- 第二阶段目标：扩展到策略回测、资金流提醒等分析能力

## 技术栈

```text
前端：Next.js + TypeScript
后端：FastAPI + Python
数据库：SQLite
图表：ECharts
UI：Tailwind CSS + shadcn/ui
部署：本地
```

技术分工：

- Next.js：页面、交互、图表、导出入口
- FastAPI：数据接口、统计逻辑、东方财富数据源适配
- SQLite：本地缓存、股票基础信息、查询历史、自选股
- ECharts：每日资金流柱状图、累计净流入折线图
- Tailwind CSS + shadcn/ui：基础 UI 与表格、表单、日期选择器等组件

## 数据库设计草案

### `stocks`

股票基础信息。

```text
code                 股票代码，例如 300308
name                 股票名称，例如 中际旭创
market               市场，例如 sz / sh
secid                东方财富 secid，例如 0.300308
industry             行业名称
updated_at           更新时间
```

### `money_flow_daily`

个股每日资金流。

```text
stock_code           股票代码
trade_date           交易日期
main_net_inflow      主力净流入
super_large_inflow   超大单净流入
large_inflow         大单净流入
medium_inflow        中单净流入
small_inflow         小单净流入
close_price          收盘价
change_pct           涨跌幅
source               数据源，例如 eastmoney
created_at           创建时间
updated_at           更新时间
```

建议唯一约束：

```text
stock_code + trade_date + source
```

### `query_history`

查询历史。

```text
id
symbols              查询股票列表，JSON 字符串
start_date
end_date
metrics              查询指标，JSON 字符串
created_at
```

### `watchlists`

自选股分组。

```text
id
name
created_at
updated_at
```

### `watchlist_items`

自选股明细。

```text
id
watchlist_id
stock_code
sort_order
created_at
```

### `fetch_logs`

数据抓取日志。

```text
id
source
target_type          stock / board / concept
target_code
start_date
end_date
status              success / failed
message
created_at
```

## 后端 API 草案

### 股票搜索

```http
GET /api/stocks/search?q=中际
```

用途：

- 支持代码搜索
- 支持名称模糊搜索
- 第一版可以先只查本地 `stocks` 表

### 资金流统计

```http
POST /api/money-flow/summary
```

请求示例：

```json
{
  "symbols": ["300308", "300502", "603986"],
  "startDate": "2026-06-08",
  "endDate": "2026-07-07",
  "source": "eastmoney"
}
```

响应示例：

```json
{
  "range": {
    "startDate": "2026-06-08",
    "endDate": "2026-07-07"
  },
  "items": [
    {
      "code": "300308",
      "name": "中际旭创",
      "mainNetInflow": -21549000000,
      "tradeDays": 21
    }
  ],
  "totalMainNetInflow": -21549000000
}
```

### 每日明细

```http
GET /api/money-flow/daily?symbol=300308&startDate=2026-06-08&endDate=2026-07-07
```

用途：

- 表格明细
- 每日资金流柱状图
- 累计净流入折线图

### 查询历史

```http
GET /api/query-history
POST /api/query-history
```

### 自选股池

```http
GET /api/watchlists
POST /api/watchlists
POST /api/watchlists/{id}/items
DELETE /api/watchlists/{id}/items/{stockCode}
```

## 数据源设计

第一版只实现东方财富，但必须预留数据源适配层。

接口抽象：

```text
MoneyFlowProvider
  search_stocks(query)
  fetch_stock_daily_flow(symbol, start_date, end_date)
  fetch_board_daily_flow(board_code, start_date, end_date)
```

东方财富实现：

```text
EastMoneyProvider
```

注意事项：

- 东方财富接口不是正式开放 API，可能出现空响应、限流、字段变化
- 请求需要设置浏览器 UA 和 Referer
- 需要记录接口失败日志
- 不要把东方财富字段名直接泄漏到前端
- 前端只使用后端标准化后的字段

## 统计口径

第一版默认口径：

```text
主力净流入 = 东方财富日线资金流中的主力净流入字段
净流出 = 主力净流入为负时的绝对值
区间净流入 = 日期区间内每日主力净流入求和
```

同时保留以下字段，方便后续扩展：

```text
超大单净流入
大单净流入
中单净流入
小单净流入
```

页面展示时建议同时显示：

- 区间净流入：保留正负号
- 净流入 / 净流出文案：根据正负号自动转换

示例：

```text
主力净流入：-215.49 亿元
方向：净流出 215.49 亿元
```

## 本地启动方式目标

最终希望做到：

```bash
cd stock-flow-lens
```

启动后端：

```bash
cd apps/api
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload
```

启动前端：

```bash
cd apps/web
npm install
npm run dev
```

访问：

```text
http://localhost:3000
```

## 实施建议

推荐实现顺序：

1. 后端先打通东方财富个股日线资金流拉取
2. 建 SQLite 表并实现缓存读写
3. 实现区间统计 API
4. 前端做查询表单和结果表格
5. 接入 ECharts 图表
6. 增加 CSV 导出
7. 再做股票名称搜索、自选股、查询历史
8. 最后做行业 / 概念板块统计和定时刷新

第一版最重要的是闭环，不要一开始追求功能完整。只要能稳定完成“多只股票 + 时间区间 + 资金流统计 + 图表 + 缓存”，这个项目就已经具备长期迭代价值。
