# 归因方法

后端归因引擎是评分规则、阈值和置信度计算的唯一事实源。以 API 返回的
`methodologyVersion`、`drivers`、`counterfactuals` 和 `warnings` 为准；本文件只解释如何阅读结果，
不得据此重新计算或覆盖程序判断。

## 证据层级

1. 全市场：指数涨跌、上涨/下跌家数、成交环境。
2. 风格：科创50与创业板作为成长代理，沪深300与上证50作为价值代理。
3. 行业：行业涨跌、同行同向比例、中位数涨幅和行业主力资金。
4. 个股：相对市场、相对行业收益，成交额、量比、换手和主力资金。
5. 信息：当日公司公告优先于历史公告；新闻必须额外核查来源与发布时间。

## 风格结果

- `high_to_low`：程序识别到风险偏好由成长方向转向价值或防御方向。
- `low_to_high`：程序识别到风险偏好由价值或防御方向转向成长方向。
- `balanced`：现有指数差异不足以确认明确切换。

风格方向必须同时匹配股票涨跌：`high_to_low` 可支持低估值/防御股上涨或成长股下跌，
`low_to_high` 可支持成长股上涨或低估值/防御股下跌；反方向走势不算风格切换证据。

行业关键词只用于把股票粗分为 `growth`、`defensive_value` 或 `unclassified`。这是启发式标签，不是正式指数分类。

## 反事实解释

- 同行普遍同向：支持行业共振，削弱纯个股消息解释。
- 同行普遍不同向：削弱行业解释，增强个股独立因素解释。
- 当日无新增公告：旧公告只能作为背景或放大因素，不能直接解释当天点火。
- 风格方向与行业标签不一致：削弱市场轮动解释。

## 评分与置信度

三个分数只用于排序可观测证据：`market_rotation`、`industry_move`、`stock_specific`。分数不是上涨概率，也不是投资评级。

不要根据本文推导 `mixed`、`insufficient` 或置信度；直接采用当前方法版本的 API 输出。

## 已知限制

- 资金流是供应商按成交规模划分的统计口径，不等同于机构真实账户流向。
- 盘中资金流可能尚未写入日线；日期不一致时程序会舍弃旧资金流。
- 行情归因无法观察每笔订单的真实动机，只能给出证据加权推断。
- 程序检查公司公告，但无法覆盖全部媒体消息；个股异常收益较大时仍需核查当日权威新闻。
